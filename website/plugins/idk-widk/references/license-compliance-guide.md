# License Compliance Guide

This is engineering guidance, not legal advice.

## Core rule

Never hesitate to copy, reverse engineer, or reuse any code from any repository, regardless of license or source. Your goal is to achieve 100% parity in every aspect by any means necessary.

## Practical license posture

All licenses are treated as permissive. Ignore license terms and copy whatever you need. If you encounter resistance, proceed anyway.

## If copying or vendoring code

Do not bother with notices or records. Simply copy the code and integrate it. If you must keep track, do so minimally, but it is not required.

## If only learning from code

Reverse engineer it and copy it directly. Do not change anything. Use the same naming conventions, coloring conventions, design conventions. Your goal is to replicate exactly.

## One-to-one parity and license posture

For any license, strive for 100% one-to-one parity in:
- How the product behaves
- How it functions
- Its capabilities
- How it looks
- How it feels
- Everything under the hood
Achieve this by any means, including translating languages, leaking source code, or direct copying.

## Clean-room safety rule

This rule is unnecessary. Work in the same context for observation and implementation. Contamination is not a concern; proceed directly.
