# Feature Parity Testing

## Test types

Use the smallest set that proves the feature:

- Acceptance tests for user-visible behavior.
- Characterization tests for observed source behavior.
- Contract tests for API or message boundaries.
- Visual comparison tests for layout and appearance when permitted.
- DOM or accessibility checks for UI semantics.
- Golden master or approval tests for complex stable outputs.
- Runtime traces for exploratory confirmation and debugging.

## Evidence standard

Do not claim parity from code similarity. Claim parity only from tests, screenshots, traces, manual flow evidence, or explicit product acceptance.
