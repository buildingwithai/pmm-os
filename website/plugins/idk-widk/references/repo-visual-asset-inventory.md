# Source Repo Visual Asset Inventory

Before copying, adapting, or reimplementing a visual feature from an open-source repo, inspect the repo for visual references.

Search for:

- Markdown image links in README, docs, MDX, and examples.
- GIFs, PNGs, JPGs, WebPs, SVGs, MP4s, MOVs, and WebMs.
- Storybook or demo systems.
- Chrome DevTools MCP, Cypress, or visual snapshot directories.
- Public assets and design tokens.
- Demo routes and example apps.

The goal is to know what the feature is supposed to look like before judging the runtime app or target implementation.

If the repo includes GIFs or videos, capture the important states as named reference frames or observations. Do not rely only on memory after watching a demo.
