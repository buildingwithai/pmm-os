# PMM Artifact Chain

Use this chain when a user wants more than advice. Each stage produces a concrete artifact that can be reviewed, shipped, or converted into another file format.

## Default chain

1. Product context
   - Product, audience, category, competitors, proof, constraints, voice, current GTM motion.
   - Output: `.agents/marketing-os/product-context.md`.

2. ICP and persona definition
   - Layered ICP, anti-ICP, buying committee, pains, language, signals, disqualification criteria.
   - Output: `icp-profile.md`, `persona-cards.md`.

3. Positioning and message architecture
   - Category, competitive alternative, differentiated capabilities, value, proof, POV, message hierarchy.
   - Output: `positioning-brief.md`, `messaging-hierarchy.md`.

4. Evidence and research synthesis
   - VOC, reviews, sales notes, win/loss, support tickets, interview notes, AEO visibility, competitor claims.
   - Output: `research-synthesis.md`, `evidence-bank.md`.

5. GTM strategy
   - Launch tier, channels, timing, owners, RACI, risk, measurement, enablement.
   - Output: `launch-brief.md`, `gtm-plan.md`, `campaign-brief.md`.

6. Activation assets
   - Blog, email, social, paid creative, sales one-pager, battlecard, demo script, website section, deck outline, image briefs.
   - Output: `asset-pack/`.

7. PMM coach review
   - Clarity, audience fit, PMM fundamentals, business impact, gaps, risks, and top improvements.
   - Output: `coach-review.md` and revised artifacts.

8. Production handoff
   - File formats, owners, approval path, MCP systems, dry-run checklist, publish checklist, measurement dashboard.
   - Output: `handoff-checklist.md`.

## Quality bar

Every artifact should answer:

- Who is this for?
- What changed?
- Why now?
- Why should the audience care?
- What proof supports the claim?
- What action should happen next?
- How will we know it worked?

If any of those are missing, run the PMM Coach loop before shipping.
