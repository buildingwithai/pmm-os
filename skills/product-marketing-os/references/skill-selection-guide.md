# PMM OS — Which Skill? (disambiguation map)

Several skills sit close together. Use this to fire the *right* one. When in
doubt, an orchestrator (`product-marketing-os`) will route for you.

## Positioning & messaging

| You want to… | Use | Not |
| --- | --- | --- |
| Create or reposition (Dunford exercise) | `pmm-positioning-exercise` | positioning-audit (that's for existing) |
| Evaluate/score positioning you already have | `pmm-positioning-audit` | positioning-exercise |
| Turn positioning into value props, message house, names, taglines, launch narrative | `pmm-messaging-positioning` | — |
| Build the formal POV → benefit → proof → feature cascade | `pmm-messaging-hierarchy` | messaging-positioning |
| Test whether messaging is *specific enough* | `pmm-message-market-fit` | — |

## Launch / GTM / orchestration

| You want to… | Use | Not |
| --- | --- | --- |
| The scoping brief (tier, workstreams, approvals) | `pmm-launch-brief` | go-to-market |
| The full GTM motion (channels, comms, measurement) — **ends with the launch kit** | `pmm-go-to-market` | launch-brief |
| End-to-end marketing/launch, multi-artifact | `product-marketing-os` | — |
| Product-management cycle (PRD, roadmap, prototype) | `product-lifecycle-os` | product-marketing-os |
| Broadest cross-system (many skills/agents/MCPs/QA) | `agentic-marketing-orchestrator` | product-marketing-os |
| Package finished work into an interactive HTML hub | `pmm-launch-kit` | artifact-factory (that makes the files) |

## The rule of thumb

**Analyze vs. create vs. evaluate**, and **artifact vs. umbrella**:
diagnose → `*-analysis`/`*-audit`; build the deliverable → the specific skill;
want the whole motion → an orchestrator. A full launch/GTM motion **always ends
by producing the interactive launch kit** (`pmm-launch-kit`).
