# PRD and Prototype Workflow

## Source

Adapted from `kelegele/oh-my-pm` PRD and prototype design patterns.

## Scenario types

| Scenario | Required context | Default artifact depth |
|---|---|---|
| Iteration update | Current feature, current UI state, iteration goal | PRD delta, acceptance criteria, prototype patch |
| New feature | Product architecture, design constraints, entry point | Full feature PRD, user flow, prototype brief |
| 0-1 product | Product context, users, constraints, reference products | Product concept, full PRD, prototype scaffold, launch implications |

## PRD sections

1. Problem and opportunity
2. Target user and use case
3. Goals and non-goals
4. User stories
5. Functional requirements
6. Non-functional requirements
7. UX flow
8. Acceptance criteria
9. Success metrics
10. Analytics events
11. Dependencies
12. Risks and open questions
13. Launch and GTM implications

## Prototype outputs

- Wireframe outline
- Mockup brief
- Interactive HTML prototype scaffold
- Design QA checklist
- Stakeholder review notes

## Handoff to GTM

Every PRD should include a small GTM implications section:

- Does this affect positioning?
- Does this change a customer workflow?
- Does this need a launch tier?
- Does sales or CS need enablement?
- Does documentation need updates?
- What metric proves customer value?
