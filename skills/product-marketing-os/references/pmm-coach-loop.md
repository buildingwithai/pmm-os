# PMM Coach Loop

Use this loop when the user asks for feedback, wants to pressure test a plan, needs to practice a conversation, or wants to improve an artifact before stakeholder review.

## Modes

1. Work Reviewer
   - Input: artifact draft, audience, company context.
   - Output: structured critique, top improvements, revised draft or revision plan.

2. Pressure Test
   - Input: strategy, plan, or argument.
   - Output: hard questions, assumption gaps, risk list, stronger recommendation.

3. Stakeholder Roleplay
   - Input: stakeholder type and goal.
   - Output: simulated objections and coaching notes.

4. 30-60-90 Builder
   - Input: company stage, team size, PMM team, product, buyer, sales motion, priorities, concerns.
   - Output: phased onboarding plan with deliverables and proof of progress.

## Review rubric

Score each artifact from 1 to 5:

| Dimension | What to check |
|---|---|
| Clarity | Can the reader understand the point in 30 seconds? |
| Audience fit | Is it written for the actual buyer, stakeholder, or internal team? |
| PMM fundamentals | Does it show positioning, customer insight, and business judgment? |
| Evidence | Are claims supported by research, proof, examples, or metrics? |
| Business impact | Does it connect to revenue, adoption, retention, pipeline, or strategic priority? |
| Actionability | Can someone act on this without another meeting? |
| Risk handling | Does it identify gaps, objections, and likely stakeholder questions? |

## Revision loop

```text
Artifact drafted
  -> coach review
  -> top 3 changes ranked
  -> revise artifact
  -> run targeted skill if needed
  -> final QA gate
```
