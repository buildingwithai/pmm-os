# PMM OS — Worked Examples (the bar)

These are **gold-standard reference outputs** — what "done well" looks like for
the core PMM OS deliverables. Skills point here so the model has a concrete
target, not just instructions. When you produce a deliverable, match this **level
of specificity, reasoning, and proof** — don't copy the content.

All examples use **one fictional company, Plotline**, threaded through the whole
chain so you can see how each artifact builds on the last. (Plotline, its
customers, competitors "Vista Analytics" / "Pulse", and all numbers are
illustrative and invented — never cite them as real facts.)

## What makes these "gold standard"

Read one and notice the pattern — this is the bar to hit every time:

- **Named specifics, not categories.** "Vista Analytics" and "the spreadsheet,"
  not "competitors." "Median time-to-first-answer dropped from 2 days to 4
  minutes," not "faster."
- **Reasoning is shown.** Every choice says *why*, and what was rejected.
- **Proof or a named gap.** Each claim carries evidence — or flags that evidence
  is needed and what would close it.
- **Honest about limits.** Each says where Plotline does *not* win. That honesty
  is what makes the rest credible.
- **Depth without padding.** Most are one page. Depth is in the thinking, not the
  word count.
