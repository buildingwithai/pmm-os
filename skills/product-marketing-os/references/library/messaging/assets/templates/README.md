# Messaging Templates

Ready-to-fill templates for creating, adapting, testing, and policing product messaging. Every template ships with purpose/when-to-use, fill-in instructions, `[bracketed]` placeholders, and a **filled Plotline example** (from the demo launch universe, `demo/plotline-launch/`) so generation targets a standard, not the mean. All written to the [deliverable standard](../../../../deliverable-standard.md); formats per the [framework registry](../../../../framework-registry.md).

## The set — in working order

**Create**
1. [`messaging-canvas-template.md`](messaging-canvas-template.md) — the main organizing canvas: primary message, pillars with typed proof, objections, channel expressions, named gaps. The working doc everything else distills from.
2. [`message-house-template.md`](message-house-template.md) — the one-page artifact that circulates: roof ≤15 words, exactly 3 pillars, 3-4 proofs each, evidence foundation, visible review date.
3. [`value-proposition-template.md`](value-proposition-template.md) — Pain-Claim-Gain per message, compressed to the four-part line: claim · who · proof · pain removed.
4. [`voice-and-tone-template.md`](voice-and-tone-template.md) — 3-4 voice attributes with we-are/we-are-not + do/don't pairs, NN/g tone dimensions, and a tone map by context (the house's "walls").

**Adapt**
5. [`persona-matrix-template.md`](persona-matrix-template.md) — message × persona mapping: each pillar rewritten as sayable copy per buyer, with per-persona objections, channels, and CTAs.

**Test & police**
6. [`message-testing-script.md`](message-testing-script.md) — the 20-25 min interview protocol with blind competitor controls, scored on `pmm-message-market-fit`'s five pillars; ship/iterate/reposition thresholds.
7. [`messaging-audit-checklist.md`](messaging-audit-checklist.md) — 9 scored rows (0-3) over your live surfaces: consistency, competitor-swap specificity, proof coverage, claim honesty, freshness. Fix the weakest row first.

## The pipeline

positioning canvas → messaging canvas → message house → value props + persona matrix

                                            └──── audit checklist (quarterly) ────┘

## Using a template

1. Copy the file; keep the section structure intact (the sections ARE the framework).
2. Replace every `[bracketed]` placeholder; delete nothing silently — an empty section is a named gap.
3. Match the filled Plotline example's evidence discipline: verbatim quotes, sourced + dated numbers, typed claims, honesty flags on anything unverified.
4. Score the result against the [deliverable scorecard](../../../../deliverable-standard.md#4-the-deliverable-scorecard-03-per-row-shippable--2433-no-row-below-2) before it circulates.
