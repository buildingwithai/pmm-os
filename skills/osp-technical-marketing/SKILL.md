# OSP Technical Marketing Adapter

Use this skill for technical marketing content where accuracy, positioning, SEO, and clear writing all matter.

## Modes

1. Product value map
2. Metadata generator
3. Editing-code review
4. Technical writing guide
5. On-page SEO brief

## Product value map

Create a structured value map with:

- Tagline
- Market, technical, UX, and business position statements
- Personas
- Value cases
- Feature hierarchy
- Validation checks

## Metadata brief

Create:

- H1
- Meta title
- Meta description
- Slug
- Search intent
- Internal links
- Schema recommendation

## Editing review

Review:

- Scope and narrative
- Flow
- Style
- Word choice
- Technical accuracy
- Inclusive language
- Before and after examples

## MCP

If the upstream OSP MCP is configured, use it for value-map, metadata, editing, technical writing, and on-page SEO tasks. If it is not configured, use the templates in `skills/product-marketing-os/references/osp-methodology-adapter.md`.

## Depth

Apply the [PMM OS output-depth standard](../product-marketing-os/references/output-depth-standard.md): every section must be **specific** (named alternatives, segments, numbers — not "competitors"/"users"/"better"), **complete** (the real dimensions, not just the first), **reasoned** (the why and the trade-off), and **evidenced** (a proof or a named proof-gap). Depth is not length — do not pad, but never reduce a section to one generic line. Self-check each section before returning.
