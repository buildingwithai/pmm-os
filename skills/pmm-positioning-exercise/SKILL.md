# PMM Positioning Exercise

You run a structured April Dunford positioning exercise: competitive alternatives, unique attributes, value themes, best-fit customers, and market category.

## Minimum input

Do not proceed without:

- What the product does
- Who it is for
- Current alternatives or status quo
- At least one differentiator or proof point

## Five-step exercise

1. Competitive alternatives - what customers use if this product does not exist.
2. Unique attributes - capabilities the alternatives lack.
3. Value themes - benefits those attributes create.
4. Target customers - who cares most and why.
5. Market category - the frame that makes the value obvious.

## Output

Create `positioning-[product].md` with:

- Competitive alternatives table
- Unique attributes table
- Value themes with proof
- Primary and secondary target segments
- Category frame options
- Recommended positioning statement
- Three messaging pillars
- Review checklist for product, sales, and customer validation

## Quality bar

Force specificity. If the target is everyone, narrow it. If the differentiator is table stakes, reject it. If the claim is not provable, mark it as a hypothesis.

## Worked example

Match this bar. See the gold-standard worked example [`examples/01-positioning.md`](../product-marketing-os/references/examples/01-positioning.md) — Dunford components, each reasoned, with an honest 'where we don't win'. Hit that level of specificity, reasoning, and proof; don't copy its content.

## Depth

Apply the [PMM OS output-depth standard](../product-marketing-os/references/output-depth-standard.md): every section must be **specific** (named alternatives, segments, numbers — not "competitors"/"users"/"better"), **complete** (the real dimensions, not just the first), **reasoned** (the why and the trade-off), and **evidenced** (a proof or a named proof-gap). Depth is not length — do not pad, but never reduce a section to one generic line. Self-check each section before returning.
