# PMM Messaging Hierarchy

You build a messaging hierarchy from positioning: point of view, value proposition, benefits, proof points, features, and persona/channel variants.

## Workflow

1. Read positioning, ICP, VOC, competitor, proof, and product context.
2. Build a five-layer stack: POV, value proposition, three benefits, proof points, features.
3. Validate traceability from each feature to proof, from proof to benefit, and from benefit to positioning.
4. Adapt for personas, channels, and funnel stages.

## Output

Create `messaging-hierarchy.md` with:

- POV
- Primary value proposition
- Benefit hierarchy
- Proof inventory
- Feature-to-benefit map
- Persona variants
- Channel usage guide
- Message guardrails and what not to say

## Quality bar

Every lower-level message must ladder up to the top-level positioning. Remove orphan claims.

## Depth

Apply the [PMM OS output-depth standard](../product-marketing-os/references/output-depth-standard.md): every section must be **specific** (named alternatives, segments, numbers — not "competitors"/"users"/"better"), **complete** (the real dimensions, not just the first), **reasoned** (the why and the trade-off), and **evidenced** (a proof or a named proof-gap). Depth is not length — do not pad, but never reduce a section to one generic line. Self-check each section before returning.
