# OSP Value Map

You build an OSP-style product value map: tagline, positioning, personas, value cases, and a feature-to-value hierarchy backed by proof.

## Workflow

1. Read product context, features, audience, proof, and technical docs.
2. Create a value map: tagline, position statements, personas, value cases, feature hierarchy, and validation checks.
3. Connect each feature to a persona need and value case.
4. Route to `osp-content-optimizer` for content pages, metadata, technical writing, and SEO.

## Output

Create `osp-value-map-[product].md` with:

- Tagline
- Market, technical, UX, and business position statements
- Personas and needs
- Value cases
- Feature hierarchy
- Proof needed
- Validation checklist

## Quality bar

No feature should exist without a value case. No value case should exist without a persona need.

PMM OS is a chain, not a menu — don't dead-end at advice. Pass the work on:

- [`osp-content-optimizer`](../osp-content-optimizer/SKILL.md) — optimize it

## Depth

Apply the [PMM OS output-depth standard](../product-marketing-os/references/output-depth-standard.md): every section must be **specific** (named alternatives, segments, numbers — not "competitors"/"users"/"better"), **complete** (the real dimensions, not just the first), **reasoned** (the why and the trade-off), and **evidenced** (a proof or a named proof-gap). Depth is not length — do not pad, but never reduce a section to one generic line. Self-check each section before returning.
